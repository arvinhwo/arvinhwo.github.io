Web视频采集

Posted on 2025-12-23 In Tool-Dev
轻量化Web视频采集工具：替代OBS/PotPlayer的极简方案

工具介绍

这是一款基于纯前端技术（HTML+CSS+JavaScript）开发的轻量化视频采集工具，无需安装客户端、无功能冗余，仅通过浏览器即可实现摄像头采集、设备切换、全屏预览等核心功能，完美适配需要极简视频采集能力的场景（如快速调试摄像头、轻量化直播推流前置采集、本地视频预览等）。

开发初衷

日常使用OBS、PotPlayer等工具进行视频采集时，常会遇到以下问题：

  1. 功能冗余：这类工具集成了录屏、推流、解码、格式转换等大量非必要功能，启动慢、占用系统资源高；
  2. 定制成本高：无法快速调整界面布局、日志展示、交互逻辑等个性化需求；
  3. 跨平台适配差：客户端工具需对应系统安装，而Web工具可在任意带现代浏览器的设备（Windows/Mac/Linux/平板）上运行。

因此开发这款轻量化Web工具，聚焦「视频采集」核心需求，保留摄像头切换、全屏预览、设备信息日志等实用功能，同时支持自定义界面样式、交互逻辑，兼顾轻量与可定制性。

核心功能

功能项 详情说明
自动摄像头初始化 页面加载后自动检测并启动默认摄像头，支持宽松/最小两种约束适配不同设备
多摄像头切换 弹窗式设备列表，支持一键切换已连接的多个视频输入设备
全屏预览 支持网页内全屏播放摄像头画面，按ESC可快速退出
实时日志输出 可视化展示设备检测、初始化、切换等全过程日志，便于调试排障
响应式布局 适配大屏/小屏设备，小屏自动调整布局为上下结构，保证操作体验
设备信息展示 输出摄像头支持的分辨率、帧率范围，以及当前实际运行参数

技术实现

核心技术栈

  • HTML：构建页面基础结构（视频容器、日志面板、设备弹窗等）；
  • CSS：通过CSS变量统一视觉风格，实现弹性布局、弹窗样式、滚动条美化等；
  • JavaScript：调用浏览器navigator.mediaDevices API实现摄像头采集，处理设备枚举、流切换、全屏交互等逻辑。

关键特性实现

  • 多设备兼容：提供「宽松约束」（理想分辨率/帧率）和「最小约束」（仅启用视频）两层初始化逻辑，适配不同品牌/型号摄像头；
  • 流管理：切换设备前自动停止当前视频流，避免摄像头占用冲突；
  • 交互优化：支持ESC键关闭弹窗/退出全屏，hover态反馈，日志自动滚动到底部等细节体验。

使用方法

快速运行

  1. 将文末代码保存为video-capture.html文件，或使用git clone git@github.com:arvinhwo/WebVedioCapture.git下载远程仓库；
  2. 用现代浏览器（Chrome/Firefox/Edge）直接打开该文件；
  3. 浏览器弹出权限请求时，允许「摄像头访问」；
  4. 页面自动初始化摄像头，即可使用各项功能。

功能操作

  • 切换摄像头：点击「切换摄像头」按钮，在弹窗中选择目标设备即可；
  • 全屏播放：点击「网页全屏播放」按钮，进入全屏模式，按ESC退出；
  • 查看日志：右侧日志面板实时展示设备状态、初始化结果、切换记录等信息

注意事项

  1. 浏览器权限：需允许摄像头访问权限，部分浏览器要求HTTPS环境（本地打开file://协议不受限）；
  2. 设备占用：使用前需关闭其他占用摄像头的程序（如微信、OBS等）；
  3. 兼容性：仅支持支持MediaDevices API的现代浏览器，不兼容IE；
  4. 流释放：关闭页面时会自动停止视频流，避免摄像头持续占用。
